A comparison of Angular to other JS frameworks like React


Differences Between Angular and React:

  • Framework vs Library
    Angular is a framework (which includes standardized pre-written code to make development faster) while React is a library (a collection of fucntions and functionalities). Code calls the library while the framework calls your code.
  • Angular
    Angular is a TypeScript-based Model View Controller (MVC) architecture used for frameworks that develop user interfaces. It was created by Google.
  • React
    React is a JavaScript library for building user interfaces. It was created by Facebook.
  • Data Binding
    Angular allows 2 way data binding, while react only allows 1 way data binding.
  • DOM Usage
    Angular uses the browser's Data Object Model (DOM), while React uses a virutal DOM, which is a simplified version of the DOM. A virtual DOM can change any element very quickly without needing to render the whole DOM.
  • Languages
    Angular is a JS framework built to use TypeScript. React is a JS library the recommends using JSX - JS with extended XML syntax.
  • Learning Curve
    TypeScript is considered more difficult to learn than JSX.
  • Native App Development Support
    There are 3 distinct versions of React: React-devtools, ReactJS.Net, and React Native. React Native allows you to build a complete mobile app in JS for either Android or iOS. Angular builds hybrid mobile apps. Native apps have better performance then hybrid mobile apps, and they look, work and feel very similar to their respective native mobile platforms.
  • App Structure
    Angular is a fully-featured MVC framework that allows applications to be divided into 3 interconnected components that are easier to manipulate for complex projects. React is more of a V in the MVC, and does not enforce an app structure.
  • When to use Angular or React
    React is ideal for apps not requiring logic or computation, such as: dynamic applications, single page applications and native mobile applications. Angular is best for creating: cross-platform mobile applications, enterprise software and progressive web applications and hybrid mobile applications. Angular in combination with Ionic is a perfect tool for buliding hybrid apps!

Why Reactive-driven forms are preferred to Template-driven forms:

  • Readability
    As a template grows, it becomes harder to follow and understand. Reactive forms are simplifed and easier to follow.
  • Visability
    It takes more code to reference the form and its controls in TDFs.
  • Validators
    Custom validators need custome directives when using TDFs.
  • Resuability
    Reactive forms are pure JS and are resuable and dynamic.
  • 2 Way Binding
    TDFs expose a 2 way binding option which breaks 1 way data flow and immutability principles, making code prone to bugs.
  • Testability
    TDFs need a component to test and use async testing. MDFs don't require such.
  • Why using both doesn't make sense
    There's a steep leanring curve, both TDFs and MDFs would need to load, it's harder to go over pull requests not knowing which is used, and it's always best to standardize.

Importance of web components:

  • Components
    A web component is a standardised way of creating encapsulated, reusable user interface elements for the web.
  • Encapsulation
    A component should be completely separate from the main application to increases reusability, testability and reliability.
  • Extensibility
    Components should be able to extend each other and in the case of web components other DOM elements.
  • Composability
    It should be possible to create more complex components or entire applications from a collection of components to decreases the need for global logic.
  • Reusability
    Components should be easily reusable with minimal dependencies and a clearly defined API.
  • Interoperability
    Components can transcend frameworks and be used in multiple projects of different technology stacks.
  • Lifespan
    Because components are interoperable they will have a longer life and there will be less need to re-write them to fit into newer technologies.
  • Portability
    Because components have little to no dependencies, the barrier to use is siginificantly lower than if components relied on a library or framework
  • Web component libraries
    Web component libraries were created to make using web components super easy, the most popular being Polymer, X-Tag and Bosonic
